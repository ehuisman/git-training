Distributed

  • All copies are equal
    • ...but some copies are more equal than others
  • (Almost) All operations are local
  • Server/client model still often used

Each commit

  • Full snapshot of the current content
  • Reference(s) to the previous commit(s)
  • Unique identifier

CVS/SVN

Git

Concepts and commands

Stages and states

  1. Working directory
  2. Index
  3. Repository

Working directory

  • Changes since the last commit
    1. Untracked files status
      • To cancel: remove the files
    2. Tracked files status, diff
      • To cancel: checkout -- [file(s)]

Index (or staging area)

  • Changes staged for the next commit add, rm
  • The same file can have both staged and non-staged changes
    • status, diff --cached
    • add --patch
  • To cancel staging: reset HEAD

Repository

  • Committed changes commit
  • Version history log
  • History can be rewritten
    rebase --interactive, commit --amend

Branches

  • Each copy is already a branch
  • Local by default
  • Easy way to jump between things
    1. branch
    2. checkout
    • checkout -b
  • In practice just a name and a reference to the latest included commit

Merge

  • Branches rarely need to live long
    1. checkout
    2. merge
    3. branch --delete

Cherry pick

  • To import individual commits from another branch cherry-pick
  • Shouldn't be need to use often

Stash

  • A temporary storage for changes which you want to store to use them later stash
  • Not part of any branch's history
  • Easy to bring back stash pop
  • Not just one change stash list
  • A branch can be created from the changes in stash stash branch

Remotes

  • When you want to share your changes with others
    • push
  • When you want to get others' changes to your repository
    • pull/rebase
  • You can have multiple remotes
